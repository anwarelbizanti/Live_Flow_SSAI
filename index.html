Live Video SSAI Workflow with SCTE-35 Markers

1. Video Source
Live encoder/camera
  • Camera/Encoder
  • Hardware: Teradek, LiveU, Tricaster
  • Software: OBS, vMix, Wirecast
↓ RTMP/SRT Stream
2. Ingest & Transcoding
AWS MediaLive / Wowza
  • Receives RTMP input stream
  • Creates ABR ladder (multiple bitrates)
  • Example: 1080p, 720p, 480p, 360p
  • Encodes to H.264/HEVC + AAC
⚡ SCTE-35 MARKER INSERTION POINT
Marker Insertion Method:
• Via API call (MediaLive Schedule API)
• Manual trigger from control room
• Automated via broadcast automation system
• Embedded in RTMP AMF metadata (legacy)
↓ HLS/DASH with SCTE-35
3. Origin/Packaging
AWS MediaPackage / Wowza
  • Packages stream into HLS/DASH
  • Maintains SCTE-35 markers in manifest
  • Provides DVR/time-shift capability
  • Creates origin for CDN distribution
#EXT-X-DATERANGE:ID="splice-001",
START-DATE="2025-10-06T10:30:00Z",
DURATION=120,SCTE35-OUT=0xFC...
↓ Manifest + SCTE-35 Markers
4A. Ad Decision Server
Google Ad Manager / FreeWheel
  • Receives ad request with viewer data
  • User location, device type, demographics
  • Decides which ads to serve
  • Returns VAST/VMAP response
  • Ad creative URLs + tracking beacons
4B. SSAI Service
AWS MediaTailor / Yospace
  • Detects SCTE-35 markers in stream
  • Requests ads from Ad Decision Server
  • Transcodes ad to match stream quality
  • Stitches ad into live stream
  • Creates personalized manifest per viewer
  • Fires tracking beacons (impressions, quartiles)
↓ Personalized HLS Manifest
5. CDN Distribution
CloudFront / Akamai / Fastly
  • Caches video segments globally
  • Reduces latency for viewers
  • Handles scale (millions of viewers)
  • Delivers both content + ads
↓ HLS Stream (Content + Ads Stitched)
6. CTV/OTT Application
Roku / Apple TV / Samsung / LG
  • Video player (native or SDK)
  • Receives seamless stream (content + ads)
  • No buffering between content and ads
  • Cannot skip ads (server-stitched)
  • Player fires additional tracking events

Key Points About SCTE-35 Markers

What SCTE-35 Markers Contain:

  • Splice command type (splice_insert, time_signal)
  • Unique event ID
  • Duration of ad break
  • PTS (Presentation Time Stamp)
  • Segmentation descriptors (program start/end, chapter, etc.)

Signal Flow:

  • Inserted at transcoding layer (Stage 2)
  • Preserved through packaging (Stage 3)
  • Detected by SSAI service (Stage 4)
  • Triggers ad decision request
  • Results in seamless ad insertion

Video Formats at Each Stage

Stage 1→2 RTMP, SRT, or RTSP (uncompressed or mezzanine)
Stage 2→3 HLS (.m3u8 + .ts segments) or DASH with SCTE-35 in manifest
Stage 3→4 HLS manifest with SCTE-35 markers (#EXT-X-DATERANGE)
Stage 4→5 Personalized HLS manifest (ads stitched in place of markers)
Stage 5→6 HLS delivery via HTTPS (video segments + audio)

SSAI Advantages

User Experience:

  • Seamless playback (no buffering)
  • Ads appear identical to content
  • Cannot be blocked by ad blockers
  • Works with DVR/time-shift
  • Consistent quality across content and ads

Technical Benefits:

  • Single player implementation
  • No client-side ad SDK required
  • Better ad completion rates
  • Personalized ads per viewer
  • Enterprise-grade reliability

Cost Considerations

AWS MediaLive + MediaPackage + MediaTailor:

  • MediaLive: ~$3-10/hour per channel (based on encoding complexity and bitrates)
  • MediaPackage: $0.04/GB ingested + $0.08/GB delivered (varies by region)
  • MediaTailor: $0.0045 per ad decision + $0.00135 per manifest generation
  • CloudFront CDN: $0.085/GB for first 10TB (decreases with volume)

Example Cost: For a 24/7 channel with 10,000 concurrent viewers, expect $5,000-15,000/month in infrastructure costs (excluding content acquisition and ad server fees).
